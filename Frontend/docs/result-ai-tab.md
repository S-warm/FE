# [FE] 결과 화면(AI 수정) 탭 구현

## Open / 배경
결과 화면에서 감지된 접근성/WCAG 및 UX 이슈를 “AI가 생성한 코드 수정안” 형태로 빠르게 확인하고,  
이전 코드 대비 변경점/영향을 한 화면에서 검증할 수 있도록 `ai` 탭을 더미 데이터 기반으로 구성합니다.

기존 결과 프레임(`/result/:simulationId/...`)과 공용 컴포넌트를 재사용해 UI를 구현하며,  
추후 API 연동 시 교체가 쉽도록 더미 데이터를 한 곳에 모읍니다.

## 목표
- 결과 탭 `ai`를 placeholder가 아닌 실제 화면으로 구현
- 좌측 패널에서 페이지 선택 가능(더미)
- “수정 할 이슈 선택”에서 이슈 카드 선택 → 코드/영향 섹션이 동기화되어 갱신
- “이전 코드” vs “AI 생성 수정 이후 코드” 비교 영역 제공
- 영향/변경 요약(무엇이 변경되었나요?) 제공
- 더미 데이터는 한 곳에 모아서(추후 API 연동 시 교체 용이)

## 작업 범위 (완료)
### Routing
- `/result/:simulationId/ai` → `ResultAiFixPage`로 연결(기존 placeholder 대체)

### AI 수정 화면(UI)
- 좌측 패널
  - 페이지 목록(더미: 로그인/메인/회원가입) + 프리뷰 placeholder
- 우측 컨텐츠
  - 헤더: “AI 생성 수정 사항” + 설명
  - 수정 이슈 카드 리스트(가로 스크롤)
  - 코드 비교: 이전 코드 / AI 수정 이후 코드(강조)
  - 영향 섹션: 개선 영향 + 변경 요약(무엇이 변경되었나요?)

### 더미 데이터
- `Frontend/src/mocks/result-ai-fix.mock.ts`에 페이지/수정안 더미 데이터 중앙화

## 결과
- 결과 화면의 `ai` 탭을 실제 화면으로 확인 가능
- 이슈 선택 → 코드/영향이 동기화되는 흐름이 갖춰져 이후 API 연동/확장에 유리
- 더미 데이터 중앙화로 추후 실데이터 연결 시 변경 범위 최소화

## 참고
- 현재 수정안/코드/영향 텍스트는 더미이며, API 스키마 확정 후 교체 예정
- 실제 적용(패치 적용/PR 생성 등) 플로우는 추후 기능 정의에 따라 추가 가능
