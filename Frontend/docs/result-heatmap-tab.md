# [FE] 결과 화면(히트맵 & 여정) 탭 구현

## Open / 배경
결과 화면에서 특정 URL의 페이지 스크린샷 위에 사용자 행동(클릭/이동/스크롤/주의)을 히트맵으로 시각화하고,  
오류 위치/세기(결함)를 함께 확인할 수 있도록 `heatmap` 탭을 더미 데이터 기반으로 구성합니다.

현재는 스크린샷을 실제 URL에서 불러오는 대신 **더미 이미지**를 사용하며,  
추후 백엔드에서 캡처 후 저장된 이미지 URL/API 연동 시 교체할 수 있도록 더미 데이터를 한 곳에 모읍니다.

## 목표
- 결과 탭 `heatmap`을 placeholder가 아닌 실제 화면으로 구현
- 좌측 패널에서 디바이스 및 페이지 선택 가능
- 우측에서 연령대/모드(클릭/이동/스크롤/주의) 선택 → 스크린샷 위 오버레이 갱신
- 결함 리스트(치명적 결함)에서 영향 사용자 수 확인 가능
- 스크린샷은 “이미지” 형태로 구성(추후 실제 URL 스크린샷으로 교체 예정)

## 작업 범위 (완료)
### Routing
- `/result/:simulationId/heatmap` → `ResultHeatmapPage`로 연결(기존 placeholder 대체)

### 히트맵 화면(UI)
- 좌측 패널
  - 디바이스(Desktop/Mobile) 토글(더미)
  - 페이지 목록(더미 4개: 로그인/메인/회원가입/결제) + 스크린샷 프리뷰
- 우측 컨텐츠
  - 연령대 선택(버튼 리스트 + 하단 슬라이더 동기화)
  - 모드 선택(클릭/이동/스크롤/주의)
  - 스크린샷 + 히트맵 오버레이(더미 좌표 기반)
  - 결함(치명적 결함) 리스트 + 영향 사용자 수 표시

### 더미 데이터/에셋
- `Frontend/src/mocks/result-heatmap.mock.ts`에 스크린샷 URL/좌표/결함 더미 데이터 중앙화
- `Frontend/src/assets/mocks/mock-page-screenshot.svg` 더미 스크린샷 이미지 추가

## 결과
- 결과 화면의 `heatmap` 탭을 실제 화면으로 확인 가능
- 스크린샷 위 오버레이(히트 포인트/마커) 기반 구조를 잡아두어, 추후 로그/API 연동이 용이
- 더미 데이터 중앙화로 추후 실데이터 연결 시 변경 범위 최소화

## 참고
- 여정(Journey)은 별도 탭(`/result/:simulationId/journey`)으로 분리
- 히트맵 포인트/마커/결함은 모두 더미이며, URL 스크린샷/로그 스키마 확정 후 교체 예정
