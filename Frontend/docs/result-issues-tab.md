# [FE] 결과 화면(주요이슈) 탭 구현

## Open / 배경
결과 화면에서 “주요 이슈”를 빠르게 탐색/검증할 수 있도록,  
기존 결과 프레임(`/result/:simulationId/...`)과 공용 컴포넌트를 재사용해 이슈 탭 UI를 더미 데이터 기반으로 구성합니다.

## 목표
- 결과 탭 `issues`를 placeholder가 아닌 실제 화면으로 구현
- 좌측 패널에서 카테고리(접근성/사용성/시각요소/기타) 필터 및 페이지 선택 가능
- 우측에서 카테고리별 분류(도넛) + 이슈 목록 확인 가능
- 각 액션 버튼은 탭 라우트로 이동되도록 연결(`ai`, `heatmap`)
- 더미 데이터는 한 곳에 모아서(추후 API 연동 시 교체 용이)

## 작업 범위 (완료)
### Routing
- `/result/:simulationId/issues` → `ResultIssuesPage`로 연결
- 이슈 카드 액션
  - “AI 수정 받기” → `/result/:simulationId/ai`
  - “히트맵에서 보기” → `/result/:simulationId/heatmap`

### 주요이슈 화면(UI)
- 좌측 패널
  - 카테고리 필터(접근성/사용성/시각요소/기타) 토글
  - 페이지 목록(더미 4개: 로그인/메인/회원가입/결제) + 펼침(프리뷰/하이라이트 배지)
- 우측 컨텐츠
  - 페이지 선택 드롭다운
  - “카테고리별 분류” 도넛 차트 + 범례(건수/비율)
  - “이슈목록” 카드 리스트
  - “이슈 전체보기” 클릭 시 필터 전체 ON + 이슈목록 영역으로 스크롤

### 더미 데이터
- `Frontend/src/mocks/result-issues.mock.ts`에 페이지/이슈 더미 데이터 중앙화

## 결과
- 결과 화면의 `issues` 탭을 실제 화면으로 확인 가능
- 이슈 탭에서 필터/페이지 기반 탐색 흐름이 가능하며, `ai/heatmap` 탭으로 라우팅 연결 완료
- 더미 데이터 중앙화로 추후 API 연동 시 교체 범위 최소화

## 참고
- 현재 이슈/페이지/분류 데이터는 더미이며, API 스키마 확정 후 교체 예정
- 페이지 프리뷰 이미지는 임시 placeholder(추후 실제 썸네일/스크린샷 연동 가능)
