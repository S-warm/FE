# [FE] 결과 화면(여정) 탭 구현

## Open / 배경
히트맵과 별도로 사용자의 “이동 경로/여정(Journey)”을 확인할 수 있도록 탭을 분리합니다.

현재는 실제 여정 로그가 없으므로 더미 데이터(좌표)로 경로를 렌더링하며,  
추후 API 연동 시 단계/노드/전환 이벤트 기반으로 교체할 수 있도록 구조를 마련합니다.

## 목표
- 결과 탭을 `heatmap`(히트맵) / `journey`(여정)으로 분리
- 여정 탭에서 페이지/연령대 선택 → 이동 경로 시각화(더미)
- 더미 데이터는 한 곳에 모아서(추후 API 연동 시 교체 용이)

## 작업 범위 (완료)
### Routing
- `/result/:simulationId/journey` → `ResultJourneyPage`로 연결
- 결과 화면 탭 네비게이션에 “여정” 탭 추가

### 여정 화면(UI)
- 좌측 패널
  - 디바이스 토글(더미)
  - 페이지 목록 + 스크린샷 프리뷰
- 우측 컨텐츠
  - 스크린샷 위 이동 경로(더미 좌표 기반) 렌더링
  - 연령대 슬라이더(더미)

### 더미 데이터/에셋
- `Frontend/src/mocks/result-heatmap.mock.ts` 재사용(추후 여정 전용 스키마로 분리 가능)

## 결과
- 히트맵/여정이 탭으로 분리되어 확장 및 기능 분리가 용이
- 여정 탭의 기본 UI/렌더링 구조를 확보해 이후 로그/API 연동 준비 완료

## 참고
- 현재 여정 경로는 더미이며, 실제 단계/노드/전환 로그 스키마 확정 후 교체 예정
