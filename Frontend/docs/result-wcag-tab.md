# [FE] 결과 화면(WCAG 검사) 탭 구현

## Open / 배경
결과 화면에서 접근성(WCAG) 준수 수준을 빠르게 확인하고,  
주요 이슈(심각도/상세 결과)를 한 화면에서 탐색할 수 있도록 `wcag` 탭을 더미 데이터 기반으로 구성합니다.

기존 결과 프레임(`/result/:simulationId/...`)과 공용 컴포넌트를 재사용해 UI를 구현하며,  
추후 API 연동 시 교체가 쉽도록 더미 데이터를 한 곳에 모읍니다.

## 목표
- 결과 탭 `wcag`를 placeholder가 아닌 실제 화면으로 구현
- 상단에서 준수 점수/통과 테스트/발견 이슈 요약 지표 확인
- “검출 이슈 분석”에서 심각도 분포 및 건수 요약 제공
- “상세 검사 결과”에서 이슈별 상세 내용을 펼쳐서 확인 가능
- 더미 데이터는 한 곳에 모아서(추후 API 연동 시 교체 용이)

## 작업 범위 (완료)
### Routing
- `/result/:simulationId/wcag` → `ResultWcagPage`로 연결(기존 placeholder 대체)

### WCAG 검사 화면(UI)
- 상단 요약 카드 3종
  - 준수 점수(%) + 기준 라벨(WCAG 2.1 LEVEL AA)
  - 통과된 테스트(통과 수 / 전체 테스트 수)
  - 발견된 이슈(총 건수) + 페이지별 breakdown(높음/중간/낮음)
- 검출 이슈 분석
  - 심각도 분포 바(Critical/Moderate/Minor)
  - 심각도별 요약 카드(건수 + 설명)
- 상세 검사 결과
  - 이슈 리스트(“자세히 보기” 토글로 내용 펼침/접기)

### 더미 데이터
- `Frontend/src/mocks/result-wcag.mock.ts`에 WCAG 결과 더미 데이터 중앙화

## 결과
- 결과 화면의 `wcag` 탭을 실제 화면으로 확인 가능
- 요약 → 분포 → 상세 이슈까지 탐색 흐름이 갖춰져 이후 API 연동/확장에 유리
- 더미 데이터 중앙화로 추후 실데이터 연결 시 변경 범위 최소화

## 참고
- 현재 지표/이슈/페이지별 breakdown은 더미이며, API 스키마 확정 후 교체 예정
- 상세 이슈의 성공기준 코드/selector/권장 수정안 등은 추후 데이터 확정 시 UI 확장 가능
